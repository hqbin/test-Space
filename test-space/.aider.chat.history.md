/**

 *

 *   - 在对应标题下追加 "- 自动记录：<commit message>"

## 自动更新本文档

为了使 `DEVELOPMENT.md` 与代码变更保持同步，可以减少手动遗漏，我们提供了一个 Node.js 脚本 `scripts/auto-update-dev.mjs`。

### 使用方法

#### 1. 自动安装 Git Hook（推荐，只需执行一次）

在项目根目录下执行：

```bash
node scripts/auto-update-dev.mjs --install
```

脚本会自动将自身注册为 `.git/hooks/post-commit`。  
之后每次 `git commit` 成功后，脚本会自动运行，根据本次 commit 修改的文件，在 `DEVELOPMENT.md` 对应的模块标题下追加一行变更记录。

> 如果 `.git/hooks` 目录不存在或尚未初始化 Git，请先确保项目中已有 `.git` 目录。

#### 2. 手动运行

```bash
node scripts/auto-update-dev.mjs
```

### 匹配规则

脚本会根据变更文件的路径匹配到 `DEVELOPMENT.md` 中的二级标题。匹配关系定义在脚本的 `MODULE_MAP` 数组中。如果文件路径没有匹配到任何已知模块，会被追加到「未归类变更」区域。

### 注意事项

- 脚本只附加简单记录（commit message + 文件数），详细的功能更新仍需手动完善。
- 安装钩子后无需再次运行 `--install`，除非 `.git/hooks/post-commit` 被删除。
- 如果 `DEVELOPMENT.md` 中已有相同模块的手动记录，脚本会在该模块下插入新行（不会删除已有内容）。
