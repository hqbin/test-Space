# ADB Web Tool

## 📖 项目简介

ADB Web Tool 是一个基于 Web 的 Android 设备管理工具，通过本地代理实现所有功能，无需后端服务器。

**核心特点**：
- ✅ 前端部署在服务器，统一更新
- ✅ 本地代理在用户电脑运行，完全后台
- ✅ 数据安全，不经过服务器
- ✅ 自动安装和更新

---

## 🚀 快速开始

### 开发者部署

#### 1. 编译本地代理

```bash
打包代理.bat
```

这将编译 Windows、macOS、Linux 三个平台的代理程序，并自动复制到前端下载目录。

#### 2. 启动前端服务

```bash
启动服务.bat
```

前端将在 `http://172.16.60.161:2333` 启动。

#### 3. 停止服务

```bash
停止服务.bat
```

### 用户使用

1. **访问网页**：`http://172.16.60.161:2333`
2. **下载代理**：首次访问时，页面会提示下载本地代理
3. **运行代理**：双击下载的程序（自动安装和配置）
4. **刷新页面**：开始使用所有功能
5. **以后**：开机自动启动，无需任何操作

详细使用说明请查看 [用户使用指南](./用户使用指南.md)

## 🏗️ 架构说明

### 部署架构

```
服务器 (172.16.60.161:2333)
└── 前端 (React 应用)
    ↓ 用户访问
用户浏览器
    ↓ 下载代理
用户电脑
└── 本地代理 (localhost:9527)
    ↓ 执行 ADB 命令
    ADB → Android 设备
```

### 核心特性

1. **前端集中部署**
   - 部署在服务器
   - 更新自动同步
   - 所有用户使用最新版本

2. **代理完全透明**
   - 自动安装到用户目录（`%USERPROFILE%\.adb-agent\`）
   - 开机自动启动
   - 完全后台运行（无界面）
   - 自动更新

3. **数据安全**
   - 代理只监听 localhost:9527
   - 外部网络无法访问
   - 设备数据不经过服务器

## 🐛 故障排查

### 问题 1：页面提示"需要本地代理"

**原因**：本地代理未运行

**解决方案**：
1. 下载并运行本地代理
2. 检查端口 9527 是否被占用
3. 查看日志：`%USERPROFILE%\.adb-agent\agent.log`

### 问题 2：无法连接设备

**原因**：设备未开启 ADB 调试或 IP 地址错误

**解决方案**：
1. 确认设备已开启 ADB 调试
2. 确认设备和电脑在同一网络
3. 确认 IP 地址正确

### 问题 3：代理无法启动

**原因**：端口被占用或 ADB 未安装

**解决方案**：
1. 检查端口 9527 是否被占用
2. 确认 ADB 已安装并在 PATH 中
3. 查看日志文件

更多问题请查看 [用户使用指南](./用户使用指南.md)
