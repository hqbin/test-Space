Authentication Flow Redesign

The current implementation relies heavily on legacy token structures. We need to transition to a more robust OAuth 2.0 flow with strict PKCE (Proof Key for Code Exchange) requirements for all mobile and desktop clients.

Key Objectives

  • Deprecate v1 authentication endpoints by Q3.
  • Implement strict JWT validation with short-lived access tokens (15m).
  • Establish robust refresh token rotation to mitigate token theft.
  • Improve error messaging for edge cases (e.g., clock skew, revoked grants).

Technical Considerations

The transition state will be delicate. We must ensure backwards compatibility during the migration window. The `AuthService` class will need significant refactoring, specifically around the `validateSession` method which currently blocks the main thread during database lookups.

I suggest moving the session cache to Redis to alleviate database load, utilizing a write-through caching strategy to ensure consistency across the distributed node pool.
